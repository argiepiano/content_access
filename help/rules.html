Introduction

This section contains notes about how to integrate with Rules.

Note: Rules integrations only work on individual nodes. The rules for access control that you set up will not be executed unless you have checked “Enable per content node access control settings”. You find this checkbox under the “Access Control” tab located on the settings page for the content type.

You can make Content Access act in much more sophisticated ways by using the Rules module. The project is fully integrated with Rules, so you can automatically set permissions for different types of content.

  • Install and the Rules project.
  • Enable both the Rules and Rules UI modules.
  • Enable the Content Access Rules Integrations

In the example we are going to show you, we have a user role called “Writer”. These people will add content to our site, but we do not want their content to be visible by everyone. We only want it to be visible to people in another user role called “Editors”. Here is how we can set this up:

  • Set up the “Writer” and “Editor” roles.
  • Navigate to Configuration » Workflow » Rules.
  • Click “Add new rule”.
